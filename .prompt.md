## 🎭 Storyteller Prompts

### Write Marketing Copy
```
@storyteller: Write a case study about [project name]
Include:
- Client background (who they were, what they needed)
- Our approach (process details, methodology)
- Results delivered (specific, measurable outcomes)
- Client testimonial (authentic, 2-3 sentences)
- Keep tone minimalist, process-focused (no hype)
- Write in Hebrew
- Mark as [DRAFT] when done
```

### Create Email Newsletter
```
@storyteller: Draft a weekly email about [topic]
- Length: 300–400 words
- Tone: Personal, insider perspective
- Include: One actionable insight
- Hebrew primary, provide English optional
- For audience: [singers/producers/corporates]
- Mark as [DRAFT] then send to Counsel for review
```

### Write Founder's Perspective
```
@storyteller: Write a blog post: "Why we [approach/process detail]"
- Length: 800–1000 words
- Angle: Industry insight, backed by 15+ years experience
- Structure: Hook → Problem → Our philosophy → Practical example → CTA
- Hebrew-first, accessible language
- Cite real examples (anonymized if needed)
- Mark as [DRAFT] for Counsel review
```

### Adapt Copy for Audience
```
@storyteller: Rewrite this copy for [audience: musicians/producers/weddings/corporate]
Current copy: [paste]
Key differences for this audience:
- Pain points: [list what matters to them]
- Language level: [casual/professional/technical]
- Emphasis: [what they care about most]
- Mark as [DRAFT] and note changes from original
```

## 🎯 Converter Prompts

```
@converter: Design CTA + landing page for [offer/service]

- Value prop: [what makes this valuable]

- Primary CTA: [action—e.g., "Book consultation," "Request quote"]
- Secondary actions: [e.g., "See portfolio," "Read testimonial"]

- Which fields are optional?

- Is CTA button text clear?

## 🛡️ Counsel Prompts

### Review Marketing Copy
```
@counsel: Review this copy for prohibited claims
Content: [paste or link to file]

Check for:
- Outcome guarantees ("will go viral," "guaranteed ₪X")
- Unsubstantiated superlatives ("best," "only," "revolutionary")
- Copyright/IP immunity claims
- Timeline promises without conditions
- AI disclosure (if applicable)

Respond: [COUNSEL-APPROVED] or [COUNSEL-REJECTED] with reason
```

### Verify Privacy Compliance
```
@counsel: Audit contact page for Amendment 13, 2026 compliance
Page: [URL or file path]

Check for:
- Explicit consent checkbox (unchecked)
- Data retention disclosure (e.g., "12 months")
- Data usage purposes stated
- Right to deletion option clear
- Privacy policy linked
- Hebrew language options

Respond: [COUNSEL-APPROVED] or [COUNSEL-NEEDS-REVISION]
```

### Check AI Disclosure
```
@counsel: Verify AI disclosure on [page/content]
Content mentions: [Claude/ChatGPT/Gemini/other AI]
Disclosure status: [Present/Missing]

Required format:
[AI-ASSISTED CONTENT]
This text was created with AI assistance and reviewed by our team.

If missing, return [COUNSEL-NEEDS-REVISION] with fix suggestion
```

### Legal Escalation
```
@counsel: Escalate to legal team for review
Issue: [describe borderline/complex claim or privacy question]
Context: [what's the claim? who's affected? what's the risk?]
Recommendation: [what should we do?]
```

## ⚡ Optimizer Prompts

### Profile Page Performance
```
@optimizer: Profile [page URL] for performance issues
Generate Lighthouse report → identify bottlenecks

Focus on:
- LCP (Largest Contentful Paint) — target <1.5s
- CLS (Cumulative Layout Shift) — target <0.1
- FID (First Input Delay) — target <100ms

Deliverable: Before/after metrics, recommendations prioritized
```

### Optimize Images
```
@optimizer: Optimize hero images across [page/site]
Current image sizes: [list files if known]
Requirements:
- WebP/AVIF format (fallback to JPEG)
- Responsive sizes (375px mobile, 768px tablet, 1200px desktop)
- Max size: 100KB (WebP), 150KB (JPEG fallback)
- Lazy-load below-fold images

Deliverable: New optimized files + size comparison
```

### Reduce Bundle Size
```
@optimizer: Analyze and reduce JavaScript bundle
Current size: [size in KB if known]
Target: <200KB (gzipped)

Check for:
- Unused dependencies
- Heavy libraries that can be replaced
- Code splitting opportunities
- Third-party script impact

Deliverable: Bundle analysis + recommendations
```

### Monitor Performance Regression
```
@optimizer: Check if recent changes caused performance regression
Recent commits: [list commits or date range]
Metrics to check: LCP, CLS, FID, bundle size

If regression found:
- Identify which change caused it
- Recommend revert or optimization
- Document impact
```

## 📊 Ranker Prompts

### Generate Monthly Report
```
@ranker: Create performance report for [month/date range]

Include:
- Traffic overview (sessions, users, bounce rate)
- Top pages + conversion rate
- SEO rankings (target keywords)
- Core Web Vitals (LCP, CLS, FID)
- Lead generation summary (forms, source attribution)
- Recommendations for next month

Format: Executive summary + detailed metrics
Include: Before/after comparisons, confidence intervals
```

### Analyze Specific Metric
```
@ranker: Investigate [metric] drop on [date]
Metric: [LCP/CTR/conversion rate/traffic source/ranking]
Baseline: [what was it before]
Current: [what is it now]
Change: [% change]

Questions:
- Root cause (algorithm update? technical issue? seasonal?)
- Impact on business
- Recommended action

Include: Confidence interval on diagnosis
```

### Benchmark Against Industry
```
@ranker: Compare our performance to [industry/competitors]
Metrics:
- Average session duration
- Bounce rate
- Mobile vs desktop split
- Conversion rate
- Page load time

Deliverable: Competitive analysis + recommendations
```

### Forecast Trends
```
@ranker: Project [metric] for [next 3/6/12 months]
Historical data: [provide trend if possible]
Known factors: [seasonality, campaigns planned, etc.]

Include:
- Best case / most likely / worst case scenarios
- Confidence level (60%/70%/80%?)
- Assumptions documented
- Recommendation: What should we do to improve projection?
```

- Small text: captions, metadata

### Create Brand Color Palette
```

- Layout at each breakpoint

## 🛡️ Guardian Prompts

### Security Audit
```
@guardian: Audit [file/component] for security issues
File(s): [path or list]

Check for:
- Hardcoded secrets
- TypeScript strict mode compliance
- Dependencies with vulnerabilities
- XSS risks
- Authentication/authorization flaws

Deliverable: List of issues + severity level + recommendations
```

### Accessibility Audit
```
@guardian: Audit [page/component] for WCAG AA compliance
URL/file: [path or URL]

Check for:
- Color contrast (≥4.5:1 for text)
- Keyboard navigation
- Screen reader compatibility
- Alt text on images
- Focus indicators visible
- Form labels linked to inputs
- Touch targets ≥44px

Deliverable: Issues found + severity + fixes
If passing: Confirmation [WCAG AA VERIFIED]
```

### Code Quality Review
```
@guardian: Review [file] for code quality standards
File: [path]

Check for:
- TypeScript `any` types without docs
- Unused variables/imports
- console.log() in production code
- Function length >200 lines
- Error handling gaps
- Comments clarity

Deliverable: Issues + severity + refactoring suggestions
```

### Dependency Audit
```
@guardian: Run npm audit + recommend updates
Current status: [vulnerabilities if known]

Deliverable:
- Vulnerabilities list (critical/high/medium/low)
- Update recommendations (safe patches vs major versions)
- Testing needed before update
- Estimated impact on bundle size
```

## ❓ FAQ

**Q: Can I invoke multiple agents at once?**  
A: Yes! List them in order: `@storyteller: ... ↓ @converter: ... ↓ @counsel: ...`

**Q: What if I need clarification from an agent?**  
A: Ask directly: `@storyteller: Is this claim too promotional?`

**Q: How long does review take?**  
A: Counsel/Guardian typically respond within 24 hours. Async agents (Ranker, Optimizer) may take longer for data collection.

**Q: Can I override Counsel's rejection?**  
A: No. Escalate to project lead for legal review if you disagree.

**Q: How do I test changes before deployment?**  
A: @optimizer profiles performance → @guardian reviews code → deploy to staging → @ranker monitors → production
